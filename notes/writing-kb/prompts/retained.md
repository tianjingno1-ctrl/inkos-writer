# 留存提示词（定稿）

> **一条记录 = 场景 + 问题 + 做法 + 认可信号**。禁止只贴 prompt 不带上下文。  
> Agent 从「这章不错」「这次改到位了」等自然反馈**自动写入**，不必等关键词。

---

## 记录模板（复制块追加）

```markdown
### [短标题] · #标签1 #标签2

**场景**  
- 书：  
- 章：  
- 任务：写章 / 润色 / 局部重写 / 整章重写 / 改对话 / 去AI味 / …  

**改之前的问题**（用户原话）  
> 

**上下文范围**  
- 发了什么：只正文 / +brief / +state / +角色卡 / …  
- 模型：Cursor / 豆包 / InkOS / …  

**做法**（用户指令 + Agent 执行的 prompt 摘要）  
> 用户：  
> Agent：  

**认可信号**（用户原话）  
> 

**再用的条件**  
- 当 … 时，套这条  

**日期**：YYYY-MM-DD  
```

---

## 避免（失败场景，同样要带场景）

```markdown
### [短标题] · 避免

**场景**：…  
**问题**：…  
**为何失败**：…  
**教训**：…  
```

## 使用说明

- 相同意图合并为一条，更新「再用条件」和日期
- 总结阶段：高频且场景清晰的条目 → 考虑写入 `.cursor/rules`
- 写新书/改稿前：Agent 先扫本书 + 类型相关标签，优先复用 retained

### 豆包润色 · 工厂文 · #番茄女频 #短语正文 #不改剧情

**场景**  
- 书：全网看我：螺丝女工干翻天才前男友  
- 任务：语感润色 / 对白感 / 炸裂点  

**改之前的问题**  
> 长约束、润色师角色会让模型改剧情、加楔子、改章标题  

**做法**  
> 用户：短语 + 正文，每次 2 章，不要 system、不要润色师  
> Prompt：`在不改变剧情的情况下，可以加一些炸裂的点或者把一些句子，描述，文风换成更适合番茄女频的，然后用对白的感觉吧`  
> 脚本：`run-doubao-polish-chapters.mjs 1 2`  

**认可信号**  
> 「可以可以，就这样！！！」  

**再用的条件**  
- 番茄女频短篇润色、改语感、加强对白时  
- 每次 2 章；自定义短语从第 4 个参数传入  

**日期**：2026-07-03
